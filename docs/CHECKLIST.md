## üìã Next Steps (Before Running)

### 1. Environment Configuration
```bash
cd hypertoq-backend
cp .env.example .env
```

Update these values in `.env`:
- [ ] `SECRET_KEY` - Generate with: `openssl rand -hex 32`
- [ ] `GOOGLE_API_KEY` - Get from Google AI Studio
- [ ] `LEMONSQUEEZY_API_KEY` - Get from LemonSqueezy dashboard
- [ ] `LEMONSQUEEZY_STORE_ID` - Your store ID
- [ ] `LEMONSQUEEZY_WEBHOOK_SECRET` - Your webhook secret
- [ ] Email settings (if using email features)

### 2. Install Dependencies

**Using Poetry:**
```bash
poetry install
```

**Using pip:**
```bash
pip install -r requirements.txt
```

### 3. Start Infrastructure
```bash
docker-compose up -d postgres redis
```

Wait for services to be healthy:
```bash
docker-compose ps
```

### 4. Run Migrations
```bash
poetry run alembic upgrade head
```

### 5. Start the Application
```bash
# Development mode
poetry run uvicorn app.main:app --reload

# Or using PowerShell (Windows)
.\commands.ps1 dev

# Or using Make (Unix/Linux/macOS)
make dev
```

### 6. Verify Installation

Open browser and check:
- [ ] API is running: http://localhost:8000
- [ ] API docs: http://localhost:8000/docs
- [ ] Health check: http://localhost:8000/api/v1/health

### 7. Test API

Try registering a user:
```bash
curl -X POST "http://localhost:8000/api/v1/auth/register" \
  -H "Content-Type: application/json" \
  -d '{"email":"test@example.com","password":"test123","full_name":"Test User"}'
```

## ‚ùì Troubleshooting

### Issue: Import errors when running tests
**Solution**: Make sure you're in the Poetry shell or using `poetry run` prefix

### Issue: Database connection refused
**Solution**: Check if PostgreSQL is running with `docker-compose ps`

### Issue: Redis connection error
**Solution**: Verify Redis is running with `docker-compose ps redis`

### Issue: Alembic can't find models
**Solution**: Make sure all models are imported in `alembic/env.py`
